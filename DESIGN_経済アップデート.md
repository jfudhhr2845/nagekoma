# 設計書: 経済アップデート（移動順・マス再編・レジャー・投資）

対象ファイル: `owan.html`（単一HTML。これが本体）
実装は本書の「実装順」どおり4段階で行い、**各段階ごとにコミット**すること。
ゲーム内の文言はすべて「〜なのだ」調・子供が読めるようにひらがな多め（既存文言に合わせる）。

---

## 0. 背景と目的

プレイテストで挙がった課題:
1. **先行有利**: 現在は総資産1位から投げ・移動する（`finishBoardRound()` のウェーバー順）。1位が先に空き物件を買える → ますます1位、の正のフィードバックになっている。「ビリは最後に投げる＝お椀で弾ける」救済は運任せで釣り合っていない。
2. **経済が不動産一本**: 収入源が物件（使用料・四半期決算・TOB）のみ。イベントマス🎲も1種類で中身が見えない。

対策（オーナー確定済みの方針）:
- ① 盤の移動・マス処理を**資産ビリから**行う（下克上しやすく）
- ② 🎲イベントマスを**経営フレーバーの4分類**に再編し、止まったら**スロット抽選演出**でイベントを決める
- ③ 🎰レジャーマス: **有馬記念**（お椀に1投して数字当て）と**スロット**（②の演出を流用）
- ④ 📈投資マス: **債券・投資信託・仕手株5銘柄**。株はニュースで乱高下。**売却は毎手番可能**

## 2. 機能①: 移動順をビリからに変更

**投げ順は現状維持**（総資産降順＝ビリが最後に投げる＝弾き放題）。**盤の移動・マス処理だけ**総資産昇順にする。

- グローバルに `let moveOrder=[0,1,2,3,4];` を追加
- `startBoardPhase()` の先頭で `moveOrder=[0,1,2,3,4].sort((a,b)=>assets(pieces[a])-assets(pieces[b]));`（このラウンド中は固定。移動中の購入で資産が変わっても並べ直さない）
- `beginPlayerMove()` と `endPlayerMove()` の `order[seqIdx]` / `order.length` を `moveOrder` に置換
- 文言更新:
  - オンボーディング「🔁 投げ順は資産ウェーバー順…」の行に「盤では**ビリから先に**動ける（空き物件を先に買える！）」を追記
  - `TUT_PAGES` の「1ラウンドのながれ」③に「ビリの会社から じゅんばんに すすむ」を反映
  - `finishBoardRound()` のコメントも実態に合わせて直す

テスト観点: ラウンド2以降、資産最下位のコマが最初にホップを始めること。1位が最初に投げること（従来どおり）。

## 3. 機能②: マス分類再編＋スロット抽選演出

### 3-1. マスの種類

`type:'event'` を廃止し、4種に分割する。28マスの割当（`EVENT_IDX` の位置を流用）:

| idx | 新type | 表示 |
|---|---|---|
| 3 | `work` | 💼 しごと |
| 7 | `leisure` | 🎰 レジャー |
| 10 | `invest` | 📊 投資 |
| 14 | `news` | 📰 ニュース |
| 17 | `hr` | 🧑‍💼 人材 |
| 21 | `work` | 💼 しごと |
| 24 | `leisure` | 🎰 レジャー |
| 27 | `invest` | 📊 投資 |

- `TILE_STYLE` に4種追加（色は既存と被らないパステル。案: work=0xe0c068 / news=0xb0a8e0 / hr=0xf0b890 / leisure=0xd888b8 / invest=0x88c8c0）。既存 start/prop/bank/tob は不変
- **`tFmt()` に新typeのチップ文言を必ず追加**（例: `→💼しごと` `→🎰レジャー` `→📊投資` `→📰ニュース` `→🧑‍💼人材`）。漏れると行き先プレビューが壊れる
- 凡例・チュートリアル「マスのみかた」を新マスで書き直す

### 3-2. イベントの再分類（EVENTS を3テーブルに分割）

`EVENTS` を `WORK_EVENTS` / `NEWS_EVENTS` / `HR_EVENTS` に分け、山札もカテゴリ別（`evDecks={work:[],news:[],hr:[]}`）にする。`drawEvent(cat,noMove)` に変更。fx処理（`applyEvent`）は既存のまま流用できる。

**WORK_EVENTS（💼しごと）= 既存の現金±30種をそのまま移す。** ただし1件だけ改名: 「株の利確でほくほく」→「とりひき先からボーナス」（機能④の株と紛らわしいため）。

**NEWS_EVENTS（📰ニュース）= 全体・業界ネタ 10種:**
既存から移動: 好景気の波(allCash+100) / 政府の給付金(allCash+80) / 業界1位から技を盗む(stealRich10%) / チャリティーに参加(payPoor100) / 地価上昇(propUp20) / リフォーム大成功(propUp30) / 地盤沈下(propDown20) / 客足がとおのく(propDown30) / 空き物件をタダでもらった(freeProp) / ボヤさわぎ！物件を手放す(loseProp)

**HR_EVENTS（🧑‍💼人材）= 移動・特殊 10種（経営フレーバーに改名）:**

| 名前 | fx | v |
|---|---|---|
| ヘッドハンティング成功！3マスすすむ | move | 3 |
| スカウトした新人が大かつやく！2マスすすむ | move | 2 |
| 天才エンジニア入社！4マスすすむ | move | 4 |
| エース社員が退職…2マスもどる | move | -2 |
| 社員旅行で羽をのばしすぎた…3マスもどる | move | -3 |
| 本社に呼びもどされた！スタートへ | warpStart | 0 |
| 新入社員が超やる気！次の出目2倍 | doubleNext | 0 |
| 敏腕交渉人をやとった！次の使用料タダ | shieldRent | 0 |
| 引きぬき合戦！だれかと場所が入れかわる | swap | 0 |
| 引きぬき成功！各社から50万 | stealAll | 50 |

連鎖制限: 既存どおり depth>0 では move/warpStart/swap を引かない（`noMove`）。人材デッキは大半が移動なので、depth1 で人材マスに止まった場合は doubleNext/shieldRent/stealAll に落ちる（それで良い）。

### 3-3. スロット抽選演出（共通コンポーネント）

DOMオーバーレイで作る（three.js不要）。**機能③のカジノスロットと共用**するため、リール数可変で設計:

```js
slotSpin({title, reels, results, onDone})
// reels: [ [ラベル配列], ... ]  リールごとの絵柄リスト
// results: [各リールの停止index]  結果は呼び出し側が先に抽選して渡す
```

- 新規 `#slotPop`（全画面オーバーレイ、中央にリール窓）。縦スクロールするラベル帯が高速回転→減速→左のリールから順に停止（停止間隔0.4s程度）。回転中は `beep()` のチック音、停止でチャイム
- phase='slotSpin' を追加。回転中のタップで即結果表示にスキップ
- **イベント抽選での使い方**: リール1本、ラベル=そのカテゴリのイベント名（山札上位10件程度で可）、結果=山札から引いた1件。停止後 `applyEvent` → 既存の `showBalloon` へ
- `landPlayer()` の分岐: work/news/hr → `drawEvent(cat)` → `slotSpin` → 適用。旧 `t.type==='event'` 分岐は削除

## 4. 機能③: 🎰レジャーマス（有馬記念・スロット）

### 4-1. 着地時の選択メニュー

汎用の選択ポップアップを1つ作る（`#resultPop` のはい/いいえでは足りない。`menuPop(title, [{label,sub,disabled,onPick}])` のような形。機能④でも使う）。

レジャーマス着地 → メニュー: 「🐎 有馬記念であそぶ（50万）」「🎰 スロットであそぶ（30万）」「やめておく」。現金不足の選択肢は disabled＋「お金がたりない…」表示。

### 4-2. 有馬記念（お椀を再利用した数字当て）

1. 賭け金50万を先払い → 数字宣言ポップ（１〜６の6ボタン、phase='gambleDeclare'）
2. **シーンをお椀に切り替え**、その社長がサイコロを**1個だけ**通常の操作（センサー/指、既存のねらい・投擲コード）で投げる。ヒント表示「🐎 ◯を出せば万馬券なのだ！」
   - グローバル `gambleCtx={p, betNum, onDone}` を追加。gambleCtx が非null の間は「ギャンブル投げモード」
   - ラウンドの既存サイコロはお椀に残ったままで良い（当たっても出目はもう消費済み。障害物として面白い）。ギャンブルダイスは `d.gamble=true` を付け、凡例・歩数計算から除外
   - `p.doubleNext` は適用しない
3. 静止・出目確定 → 盤シーンに戻る → 結果バルーン:
   - 的中: **+300万**（純益+250万）「🎯 大的中！万馬券 +300万なのだ！」
   - はずれ: 「はずれ…50万がとんでいった…」
4. `afterLand(p)` に合流（reasonsRound に「馬券+300」/「馬券-50」を積む）

期待値 = 1/6×300−50 = ±0（公平・高分散）。シーン切替は既存の お椀⇔盤 切替処理（`backToBowl`/`startBoardPhase` 周辺のシーン・カメラ・HUD切替）を関数化して流用すること。

### 4-3. スロット（§3-3コンポーネント流用）

- 参加費30万先払い → 結果を先に抽選 → `slotSpin` リール3本、絵柄 `['7️⃣','🍒','🔔','⭐','🍀']`
- 配当テーブル（結果に合わせてリール停止位置を組む）:

| 役 | 確率 | 配当 |
|---|---|---|
| 7️⃣×3 | 5% | +300万 |
| 7以外の3揃い | 10% | +150万 |
| 2揃い | 25% | +60万 |
| ばらばら | 60% | 0（参加費のみ） |

期待値 +15万/回（ちょい甘。テストプレイで渋くする余地あり）。結果バルーン→`afterLand`。

## 5. 機能④: 📊投資マス（債券・投信・仕手株）

### 5-1. 商品と市場データ

```js
const BOND_PRICE=100, BOND_COUPON=5;          // 債券: 毎ラウンド+5万/枚、価格固定、いつでも100万で売れる
let fundPrice=100;                             // 投信: 毎ラウンド ×(0.97〜1.08) で微変動（期待+2.5%）。配当なし
const STOCKS=[                                 // 仕手株5銘柄（配当なし・乱高下）
  {name:'キノコバイオ製薬', emoji:'🍄', price:100},
  {name:'ドングリコイン',   emoji:'🪙', price:60},
  {name:'ハニーゴールド鉱業',emoji:'⛏️', price:120},
  {name:'モリモリ宇宙電力', emoji:'🚀', price:150},
  {name:'リスカーゴＨＤ',   emoji:'📦', price:80},
];
const STOCK_NEWS=[   // 15%/銘柄/ラウンドで通常変動の代わりに発動
  {h:'好決算サプライズ！',     m:1.2}, {h:'大型ていけいが発表！', m:1.4},
  {h:'新技術の開発に成功！',   m:1.6}, {h:'ストップ高れんぱつ！', m:1.8},
  {h:'リコールで大炎上…',     m:0.7}, {h:'社長が雲がくれ…',     m:0.6},
  {h:'粉飾決算が発覚…',       m:0.5}, {h:'上場はいしの危機！',   m:0.3},
];
```

- 保有はプレイヤーに持たせる: `p.bonds=0; p.funds=0; p.shares=[0,0,0,0,0];`（`buildBoardPieces`/リセット処理で初期化）
- 価格更新は `roundSummary()` の冒頭で毎ラウンド1回:
  - 投信: `fundPrice=Math.round(fundPrice*(0.97+Math.random()*0.11))`
  - 各株: 15%でニュース（`price=Math.max(10,Math.round(price*m))`、見出しを記録）、85%で `×(0.80+Math.random()*0.45)`。下限10万
  - 債券利息: `p.cash+=p.bonds*BOND_COUPON`、reasonsRound に「利息+n」
- **`assets()` を更新**: `現金＋物件＋p.bonds*100＋p.funds*fundPrice＋Σ p.shares[i]*STOCKS[i].price`。チュートリアル・helpPop の「総資産＝現金＋物件」の説明文もすべて「＋株や債券」に更新

### 5-2. 投資マス着地（買い。1回の着地で1つだけ購入）

メニュー（§4-1の `menuPop` 流用）:
「🏦 債券を買う（100万・毎月+5万）」「📊 投資信託を買う（100万・じわじわ値上がり）」「🚀 株を買う（一発逆転！）」「やめておく」
→ 株を選ぶと5銘柄のサブメニュー（絵文字＋名前＋現在値）。現在値で1株購入。現金不足はdisabled。購入後バルーン「🚀 ドングリコインを60万で買った！」→ `afterLand`。

### 5-3. 売却（毎手番）

- `showTurn()`（お椀・投げる前）で、その社長が何か保有していれば画面隅に「📊 もちもの」ボタンを表示（phase='await' 中のみ反応）
- タップでポートフォリオポップ: 保有ごとに1行「🍄キノコバイオ製薬 ×2　現在値100万 〔１株売る〕」。売るたび現在値でcashに加算・即バルーン級の小フィードバック（beep＋行更新）。閉じると投げに戻る
- 損益表示（買値管理）はV1では不要。現在値と直近変動の▲▼だけで良い

### 5-4. 株式市況の発表（roundSummary に追加）

収支発表ポップの先頭に市況セクションを毎ラウンド表示:

```
📈 森の株式市場
🚨 ドングリコイン「粉飾決算が発覚…」-50%！   ← ニュースがあった銘柄のみ
🍄100→120 🪙60→30 ⛏️120→118 🚀150→180 📦80→77   ← 5銘柄コンパクト1〜2行
```

ニュース見出しは持っていない社長も巻き込む演出の核なので、**目立たせる**（色付き・🚨/📈絵文字）。

## 7. 実装順とコミット分割

1. **機能①（移動順）** — 小さく即効。コミット
2. **機能②（マス再編＋スロット演出）** — EVENTS分割・tiles生成・tFmt・slotSpin・文言。コミット
3. **機能③（レジャー）** — menuPop・有馬記念（シーン切替の関数化含む）・スロット役。コミット
4. **機能④（投資）** — 市場データ・売買UI・assets拡張・市況発表。コミット

各段階の後に:
- `node --check`（script抜き出し）＋ 未定義識別子スキャン
- 通しプレイ確認（デバッグパネルの早送り等を活用）: 12ラウンド完走・ゲーム終了→リセット→再スタートで市場価格と保有が初期化されること
- リセット処理（`resetAll` 検索）に新規状態（moveOrder/evDecks/gambleCtx/fundPrice/STOCKS価格/保有）の初期化を忘れない
